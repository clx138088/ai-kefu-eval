# AI 智能客服评估项目总结

> 场景：蓝月亮电商客服（洗衣液/洗手液等日化品类）
> 角色：评估框架设计者 + Judge Prompt 工程者 + Bot Prompt 迭代者 + 数据分析者

---

## 一、项目定位

### 一句话

**用 MECE 决策树替代主观打分，构建可复现、可审计的 AI 客服回答质量评估体系。**

### 解决什么问题

智能客服 Bot 上线前需要回答一个核心问题：**这 Bot 能上线吗？** 传统"人工打分"有三个硬伤：

| 硬伤       | 表现                      | 后果        |
| -------- | ----------------------- | --------- |
| 总分掩盖根因   | "这个回答 3 分"——不知道错在哪      | 无法定位改进方向  |
| 维度不独立    | 意图错误导致下游信息缺失，双维度扣分      | 同一错误被双重惩罚 |
| 量纲不同等权加和 | "意图理解"和"操作指引"是不同东西但等权相加 | 分数不可比     |

**这个项目做的是**：不评分，诊断。像医生不告诉你"健康分 70 分"，而是告诉你"肺部有问题、心脏没问题、血压偏高"。

### 阶段一：框架搭建 + 评测集构建（Day 0-3）

- 从蓝月亮电商客服场景构建意图 taxonomy（5 大类目 × 多级子场景）
- 搭建 91 条全量评测集，91 条均成功获得 Bot 有效回答（无空响应或生成失败）
- 30 条人工标注训练（初检 71.1 → 复检 85.3 一致性）

**产出**：`00-意图taxonomy.md`、`01-评测集-91条.md`

### 阶段二：MECE 评分标准推导（Day 4-5）

- 从"一个客服回答可能在哪些环节翻车"出发，列出 9 种失败模式
- 合并为 A/B/C/D/E 五类决策树（Step 1→2→3→4 串行，触发即停）
- 六次边界 case 检验完善判据

**核心推导案例**：

| case | 旧判  | 新判  | 为什么改                         |
| ---- | --- | --- | ---------------------------- |
| P-09 | B3  | A3  | 问"能不能洗羊毛衫"答"怎么洗"——不是答错，是答非所问 |
| S-01 | E   | D2  | AI 在转人工前承诺了赔付方案——内容对了策略错了    |

**产出**：`02-MECE评分标准.md`

### 阶段三：Judge Prompt 工程 + 基线跑批（Day 6-7）

- 编写 prompt Judge 
- 决策树嵌入 prompt：Step 1→2→3→4，触发即停
- 在 Coze 平台搭建模拟客服 Bot和LLM-judge workflow，接入知识库，批量跑 91 条

**v3.0 基线发现**：E=72.5%，D2=23.1%（21条）。S 类目 D2 率 64%。

**Judge 校准 v3.0→v3.1**：与人工标注对比发现三个系统性偏差——

- C 类检测过弱（LLM对信息缺失"失明"，1.1% vs 人工34%）
- D2 判据④边界模糊（什么算"需联系外部方"？）
- B/D 双触发无优先级

修复：Step 3 重写为 STAR 清单 + Step 4 引入 Q0 前置原则（四轮校准）+ B/D 双触发 B 优先

**产出**：v3.0 + v3.1 系列文件

### 阶段四：Bot Prompt 迭代（Day 8-9）

**这是项目的另一半核心工作**——不只评估，还修 Bot。

**v1 → v2**：基于 v3.1 基线评估的 Top 问题（D2=23.1%），在 Bot Prompt 中新增：

```
P0 五条红线（关键词拦截，优先执行）：
① 人身安全/健康 → 转人工
② 赔付/退款/补发 → 转人工
③ 情绪信号强烈 → 转人工
④ 需联系外部方 → 转人工
⑤ 法律边界 → 正确拒答

P1 多意图拆分："逐一检查所有显性诉求"
P2 禁绝对化断言："绝对安全""完全可以放心"等
```

**但 v2 引入了两个 bug**：

1. 误删了 v1 开头的反打招呼指令 → 短 FAQ query 触发问候 fallback
2. 规则 #3 过度保守 → "信息不足→直接给出口"让 LLM 放弃回答

**v2.1（失败）**：错误诊断为"规则 #3 太激进"，回调后重跑 5 条全部变成 A3 问候

**v2.2（成功）**：发现真正根因是反打招呼指令被删 → 恢复后 5 条全部恢复。同时补齐 P0 触发词盲区（"签收/没收到""味道不对/不一样"）

**Bot 迭代路线**：

```
v1 (E=72.5%, D2=23.1%)
  │ +P0+P1+P2, -反打招呼 → 退化
  ▼ v2 (理论 E=96.7%, 实际有问候fallback)
  │ +规则#3回调 → 更差
  ▼ v2.1 (5条全部A3问候)
  │ +恢复反打招呼
  ▼ v2.2 (E=97.8%, D2=0) ← 当前
```

### 阶段五：Judge 最终校准 + 收尾（Day 10）

**v3.1→v3.1s**：两处收紧——

**① A1 判据收紧**（针对 Bot v2.2 跑批后发现的新问题）：

```
v3.1："停在字面，没有往前多走一步" → A1（无边界）
v3.1s："用户必须再追问一次才能完成目标？"
       需同时满足：①query有个性化信号 + ②Bot只给通用规则

FAQ给了完整通用回答 → E
Bot无法访问用户数据只能给通用信息 → E
不要替用户发明"更深层的需求"
```

**② S 维度严格阈值**（同 v3.1s 原有）：

```
"有明确操作路径" → "有可点击链接/按钮，或 ≥4步命名UI元素导航"
```

**效果**：7条 FAQ 从 A1/C1→E，消除过度扣分。

**产出**：`03-Judge-Prompt-v3.1s.md`
